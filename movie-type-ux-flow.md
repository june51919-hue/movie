---

## 1. 서비스 개요

### 핵심 컨셉
> "넌 어떤 영화 사람이야?"

MBTI처럼 간결하고 공유 가능한 **영화 성향 유형**을 부여한다.
테스트 결과는 단순 추천에서 끝나지 않고, **정체성 콘텐츠**로 기능한다.

### 핵심 가치 (Value Proposition)
| 사용자 Pain | CINETYPE 해결 방식 |
|---|---|
| 넷플릭스 추천이 뻔함 | 알고리즘이 아닌 성향 기반 큐레이션 |
| 뭘 볼지 결정 못 함 | 테스트가 선택의 근거를 줌 |
| 영화 취향을 설명하기 어려움 | 유형 카드로 타인에게 공유 가능 |
| SNS에 올릴 콘텐츠 부족 | 결과 카드가 공유 자체가 됨 |

---

## 2. 사용자 유형 정의 (Persona)

### Primary User: 젠지 영화 탐색자
- 연령: 18–27세
- 행동 패턴: OTT 2–3개 구독 중, 뭘 볼지 30분 이상 탐색하다 포기 경험 있음
- 동기: "나랑 취향 맞는 영화" 발견, SNS 공유 욕구
- 저항 요인: 긴 설문, 회원가입 강요, 뻔한 결과

### Secondary User: 커플 / 친구 그룹
- 상황: 같이 볼 영화 고르는 중
- 동기: 의견 충돌 없이 합의된 선택
- 활용: 각자 테스트 후 결과 비교

## 3. 영화 성향 유형 설계

### 유형 분류 축 (2×2×2 = 8가지)

```
축 A. 세계관:  [현실 (R)] vs [판타지/SF (F)]
축 B. 감정:    [감성/드라마 (E)] vs [자극/스릴 (T)]
축 C. 시선:    [군상/사회 (S)] vs [개인/내면 (I)]
```

### 8가지 유형표

| 코드 | 유형명 | 설명 | 대표 영화 |
|---|---|---|---|
| REI | **현실 감성 독백러** | 잔잔하고 내면을 파고드는 영화 | 비포 선라이즈, 패터슨 |
| RES | **현실 감성 관찰자** | 인간군상을 따뜻하게 담은 영화 | 기생충, 카모메 식당 |
| RTI | **현실 스릴 내면파** | 심리 스릴러, 반전 있는 현실 | 블랙스완, 나이트크롤러 |
| RTS | **현실 스릴 사회파** | 사회 고발, 범죄 현실 | 조디악, 스포트라이트 |
| FEI | **판타지 감성 몽상가** | 서정적인 판타지, 애니 | 센과 치히로, 이터널 선샤인 |
| FES | **판타지 감성 세계관러** | 방대한 세계관, 서사 중심 | 반지의 제왕, 듄 |
| FTI | **판타지 스릴 다크너** | 다크 판타지, 복잡한 주인공 | 다크나이트, 에이리언 |
| FTS | **판타지 스릴 액션파** | 박진감, 스펙터클 중심 | 매드맥스, 아바타 |

### SCREEN 01 — 랜딩 페이지

**목적:** 테스트 시작 유도 / 서비스 인상 형성

**레이아웃 구성:**
```
┌────────────────────────────────────┐
│  [배경: 영화 스틸컷 콜라주 - 움직임]│
│                                    │
│      CINETYPE                      │
│   나는 어떤 영화 사람일까?         │
│                                    │
│   7가지 질문으로 알아보는          │
│   나의 영화 성향 유형              │
│                                    │
│   ┌──────────────────────────┐    │
│   │   테스트 시작하기  →     │    │
│   └──────────────────────────┘    │
│                                    │
│   📊 오늘 42,891명이 테스트했어요  │
│                                    │
│   [최근 공유된 결과 카드 슬라이더] │
└────────────────────────────────────┘
```

**인터랙션:**
- 배경: 유명 영화 스틸컷이 천천히 패럴랙스로 흐름
- CTA 버튼 hover 시 색상 반전 + 미세 진동 애니메이션
- 최근 결과 슬라이더: 실제 유저 결과 카드 자동 스크롤 (사회적 증거)
- 회원가입 없이 바로 시작 가능 (진입 장벽 최소화)

**이탈 케이스:**
- 다른 유저 결과 카드 클릭 → 해당 유형 미리보기 모달 → "나도 테스트하기" 유도

### SCREEN 02 — 질문 플로우 (Q1–Q7)

**목적:** 성향 데이터 수집 / 몰입감 유지

**진행 방식:**
- 전체 7문항, 한 번에 1문항씩
- 상단 진행 바 (Progress Bar) 표시
- 이전 문항으로 돌아가기 가능

**질문 구조 설계:**

```
Q1. 세계관 파악 (R vs F)
──────────────────────────────
"영화 속 세계는 어떤 느낌이길 좋아해?"

  A) 지금 우리 삶과 가까운 이야기    → R
  B) 현실에 없는 세계로 완전히 탈출  → F

──────────────────────────────
Q2. 감정 결 파악 (E vs T)
──────────────────────────────
"영화 보고 나서 어떤 감각이 남아있으면 좋아?"

  A) 마음 어딘가가 울림              → E
  B) 심장이 쫄깃했던 느낌            → T

──────────────────────────────
Q3. 시선 파악 (S vs I)
──────────────────────────────
"주인공이 이런 캐릭터였으면 좋겠어:"

  A) 세상과 부딪히는 사람            → S
  B) 자기 자신과 싸우는 사람         → I

──────────────────────────────
Q4. 감정 깊이 (E/T 강도)
──────────────────────────────
"영화 보다가 울어본 적 있어?"

  A) 자주 울어, 그게 좋아서 봄       → E 강화
  B) 가끔 멍하게 있다가 나중에 옴    → 중립
  C) 거의 안 울어, 생각이 더 많아짐  → T 강화

──────────────────────────────
Q5. 복잡도 선호 (세계관 깊이)
──────────────────────────────
"이런 영화가 더 땡겨:"

  A) 보는 동안 완전히 몰입되는 영화  → F 강화
  B) 보고 나서 계속 생각나는 영화    → R 강화

──────────────────────────────
Q6. 서사 vs 연출 선호
──────────────────────────────
"이 장면에서 더 심장이 뛸 것 같아?"

  [이미지 A: 클라이맥스 액션 장면]   → T 강화
  [이미지 B: 두 사람의 침묵 장면]    → E 강화

──────────────────────────────
Q7. 소셜 vs 개인 감상
──────────────────────────────
"영화는 보통 어떻게 봐?"

  A) 누군가랑 같이 보고 얘기하는 게 좋아  → S 강화
  B) 혼자 조용히 내 페이스로             → I 강화
```

**화면 레이아웃:**
```
┌────────────────────────────────────┐
│  ██████████░░░░░░░░░  Q4 / 7      │
│                                    │
│  "영화 보다가 울어본 적 있어?"     │
│                                    │
│  ┌──────────────────────────────┐ │
│  │  😭  자주 울어, 그래서 봄    │ │
│  └──────────────────────────────┘ │
│  ┌──────────────────────────────┐ │
│  │  😶  나중에 갑자기 옴        │ │
│  └──────────────────────────────┘ │
│  ┌──────────────────────────────┐ │
│  │  🤔  거의 안 울고 생각이 많음│ │
│  └──────────────────────────────┘ │
│                                    │
│  ← 이전               다음 →      │
└────────────────────────────────────┘
```

**인터랙션 상세:**
- 선택지 선택 시 → 0.3초 후 자동으로 다음 질문으로 넘어감 (빠른 흐름)
- 선택 후 선택지에 체크 표시 + 색상 강조
- 질문 전환 시 슬라이드 또는 페이드 전환
- Q6처럼 이미지 기반 선택지는 카드 형태로 제공

- 성향 레이더 차트 (3축 시각화)
- 나와 잘 맞는 유형 / 안 맞는 유형 (궁합)
- 이 유형에 속하는 유명인 (ex. "팀 버튼 감독도 이 유형")
- 대표 추천 영화 3편 (포스터 + 한 줄 이유)

## 6. 엣지 케이스 & 예외 처리

| 상황 | 처리 방식 |
|---|---|
| 테스트 중 뒤로가기 | "진행 중인 테스트가 있어요. 이어할까요?" 모달 |
| 결과가 경계선 (점수 동점) | 두 유형 혼합형으로 표시 (ex. "FEI/FES 혼합형") |
| 재테스트 요청 | "다시 해볼래?" CTA 제공, 이전 결과와 비교 |
| 링크 공유 후 미접속 친구 | 7일 유효 링크, 만료 시 재발급 안내 |
| 오프라인 상태 | 질문 데이터 로컬 캐싱, 결과 제출은 온라인 복귀 후 |

## 8. 기술 구현 고려사항

### 프론트엔드
- 질문 전환: CSS transition + requestAnimationFrame으로 부드러운 슬라이드
- 레이더 차트: Chart.js 또는 D3.js
- 결과 카드 이미지 생성: html2canvas로 클라이언트 사이드 렌더링
- 공유 카드: Canvas API로 커스터마이징 후 blob 다운로드

### 데이터
- 유형 판정 로직: 7문항 가중치 합산 → 3축 점수 계산 → 유형 코드 결정
- 결과 저장: localStorage (비로그인) / DB (로그인 시)
- 친구 비교 링크: UUID 기반 단기 세션 토큰

### 성능
- 질문 이미지 사전 프리로딩 (Q6 이미지 등)
- 결과 카드 폰트 임베딩으로 공유 이미지 일관성 유지
